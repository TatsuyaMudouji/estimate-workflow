# estimate-workflow  

## 概要
既存の見積ロジック・サンプルコードをベースに、 業務で再利用可能な見積フローへ再構成するための検証リポジトリ。  

## このリポジトリの目的 
- 他人が書いたコードを理解・分解できること
- 業務要件に合わせて再設計できること
- 再利用前提の構造に落とし込めること

## 採用した理由 
- 見積業務に近い構造を持っていたため
- ロジックが単純で改修余地があったため

## 改修方針 
- UIと計算ロジックの分離
- 業務前提の入力制限を追加
- 他案件に流用できる構造へ整理

## 想定ユースケース 
- 小規模業務システムの見積補助
- 概算シミュレーション

## 既存実装からの再設計ポイント
- 業務ロジックをUIから完全に分離し、単体で再利用可能にした
- UIは最小構成とし、業務ルールへの依存を排除した
- 将来的にAPI化・別UI接続が可能な前提で設計している

## Python
- Python: 見積ルールの検証用（UI非依存のロジック確認）
